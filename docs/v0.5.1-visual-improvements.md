## Design principles

Before any implementation, internalize these:

1. **Scanner-first.** If someone stares at the report for 5 seconds, they must walk away with: visibility health, trend direction, top action. Every improvement below serves that goal.
2. **Honest visualization.** No inflated colors, no "success green" at 0% score. If invisible, show invisible — emotionally neutral but factually clear.
3. **Zero new deps.** Everything inline SVG + markdown. Same palette: `#10b981` green / `#ef4444` red / `#f59e0b` yellow / `#94a3b8` gray. Add one accent: `#6366f1` indigo for "YOU" highlighting. Never brand colours.
4. **Graceful degradation.** Single-run report must look good. Multi-run must look better. Don't design only for week 12.
5. **All text in English.** Report consumers are global; implementing agent and user ops are in Russian, but output stays English.

## P1 — Hero card + traffic light

**Goal:** first screen of report answers 3 questions in 5 seconds: How am I doing? What changed? What do I do?

**File:** `lib/report/sections.js` (new export `sectionHero`)

**Placement:** first section after title/period. Before "Report in 5 Numbers" (which stays as secondary detail).

### Traffic light thresholds

| Score range | Status | Color |
|---|---|---|
| `0%` | `INVISIBLE` | `#ef4444` (red) |
| `1–25%` | `EMERGING` | `#f59e0b` (orange) |
| `26–60%` | `PRESENT` | `#f59e0b` (yellow — slightly lighter) |
| `61–100%` | `STRONG` | `#10b981` (green) |

For multi-run reports, also compute trend vs previous:
- Delta ≥ +5pp → `▲ +Npp`, green
- Delta ≤ -5pp → `▼ -Npp`, red
- Abs delta < 5pp → `— stable`, gray
- First run → `BASELINE`, gray

### Hero SVG layout (pseudocode)

```
<svg viewBox="0 0 640 180" width="100%">
  <!-- background panel -->
  <rect x="0" y="0" width="640" height="180" rx="12" fill="#f8fafc"/>

  <!-- Status badge top-right -->
  <rect x="480" y="24" width="130" height="30" rx="15" fill="{statusColor}"/>
  <text x="545" y="44" text-anchor="middle" fill="#fff" font-weight="700">
    ● {STATUS}
  </text>

  <!-- Big score number left -->
  <text x="32" y="110" font-size="64" font-weight="800" fill="#0f172a">
    {score}%
  </text>

  <!-- Trend indicator right of score -->
  <text x="190" y="72" font-size="16" fill="{trendColor}">
    {▲/▼/—} {delta}pp
  </text>
  <text x="190" y="96" font-size="13" fill="#64748b">vs previous run</text>

  <!-- Subtitle -->
  <text x="32" y="150" font-size="14" fill="#475569">

  </text>

```

### Plain-text fallback after SVG

Two markdown lines below the SVG for accessibility + text-search:

```markdown
**Visibility: {score}% — {STATUS}** · {mentions}/{total} mentions · {trend text}
```

### Acceptance

- Single-run 0% case shows: big `0%`, `INVISIBLE` red badge, `BASELINE` gray label
- Multi-run +12pp case shows: big score, `▲ +12pp` green
- Multi-run -8pp case shows: big score, `▼ -8pp` red

## P4 — Executive Summary block

**Goal:** auto-generated 2–4 sentence plain-English summary immediately after hero card. A skimmer can read only this and understand the week.

**File:** `lib/report/sections.js` (new export `sectionExecutiveSummary`)

**Placement:** after hero (P1), before "Report in 5 Numbers".

### Generation logic (no LLM call — pure template)

Input: latest snapshot + previous (if any) + topCompetitors + topCanonicalSources.

```javascript
function buildExecutiveSummary(latest, previous, disambig) {
  const parts = [];

  // Sentence 1 — visibility state
  if (latest.score === 0) {
    parts.push(`Your brand is **invisible across all ${latest.total} checks** this run.`);
  } else if (latest.score < 25) {
    parts.push(`Your brand is **emerging** — ${latest.mentions}/${latest.total} checks mentioned you this run.`);
  } else {
    parts.push(`Your brand is **present in ${latest.score}% of checks** (${latest.mentions}/${latest.total}).`);
  }

  // Sentence 2 — trend (if previous exists)
  if (previous) {
    const delta = latest.score - previous.score;
    if (Math.abs(delta) < 5) {
      parts.push(`Visibility is **stable** vs previous run.`);
    } else if (delta > 0) {
      parts.push(`Visibility **grew ${delta}pp** vs previous run.`);
    } else {
      parts.push(`Visibility **dropped ${Math.abs(delta)}pp** vs previous run — regression flagged.`);
    }
  } else {
    parts.push(`This is your **baseline run** — trend analysis activates from run 2 onward.`);
  }

  // Sentence 3 — competitive picture
  const comps = latest.trackedCompetitors || [];
  if (comps.length > 0) {
    const top = comps.slice(0, 3).map(c => `${c.name} (${c.count}×)`).join(', ');
    parts.push(`Competitors seen instead: **${top}**.`);
  }

  // Sentence 4 — disambiguation hint if triggered
  if (disambig) {
    parts.push(`⚠ Possible query/industry mismatch detected — see warning below.`);
  }

  return parts.join(' ');
}
```

### Render

```markdown
## Summary

{joined sentences as single paragraph}
```

### Acceptance

- 0-score run produces: "invisible across all X checks... baseline run... competitors seen instead: Y..."
- Stable run: "present in 30%... visibility stable... competitors..."
- Regression run: "visibility dropped 15pp — regression flagged..."
- All sentences grammatically correct across score ranges

## P6 — Actions checkbox card

**Goal:** 1–3 concrete actions in a visually distinct block user can copy-paste into todo app. Not buried in narrative.

**File:** `lib/report/sections.js` (new export `sectionActions`)

**Placement:** after Executive Summary, before Matrix. High prominence.

### Action generation heuristics

Actions are derived from signals in the report — no LLM call:

| Signal | Action |
|---|---|
| Score 0% + disambiguation warning triggered | `Re-run init with tighter --category="<specific>"` |
| Score 0% + no disambiguation | `Manual browser test on Perplexity + Copilot (they often see brands API doesn't)` |
| Score 0% + brand NPM package listed on site | `Check npm/GitHub stars — training-data signal growth over 6-12 months` |
| Top competitors share a common URL | `Backlink outreach: get listed on {URL}` |
| One engine scores 100% others 0% | `Deepen content where the working engine sources from` |
| Previous run had mention, now lost | `Investigate lost mention: diff against last run's raw responses` |
| Score 50–80% stable | `Refresh keywords: init --refresh-keywords to uncover new queries` |
| First run baseline | `Plan weekly cadence: cron or manual reminder — trends begin run 2` |

If none match → fallback: `Run weekly. Track for at least 4 runs before interpreting trend.`

### Render

```markdown
## ☐ Actions this week

- [ ] Check Perplexity and Copilot manually — they often detect brands that APIs miss
- [ ] Audit canonical sources — directory submissions to {top-source domain}
- [ ] Plan weekly cadence: trends begin from run 2
```

Checkboxes are literal markdown `- [ ]` syntax — rendered as unchecked checkboxes in GitHub, VS Code preview, etc.

### Acceptance

- Max 3 actions, min 1 action
- Each action: <80 chars, concrete verb + specific target
- Copy-pastable into Linear/Todoist as task list
- Actions change based on signals (not hardcoded)

## P2 — Radar chart: engine-by-engine coverage

**Goal:** single image showing score per engine on polar axes. Glance-level competitive coverage picture.

**New file:** `lib/svg/radar.js` (new SVG primitive)

**File to wire:** `lib/report/sections.js` (new export `sectionEngineRadar`)

**Placement:** after Matrix heatmap. Complements heatmap with different view.

### radar.js primitive

```javascript
/**
 * Render a radar (spider) chart with N axes.
 *
 * @param {Array<{label: string, value: number}>} axes  values 0-100
 * @param {object} opts
 * @param {number} [opts.size=320]    viewBox side in pixels
 * @param {number} [opts.max=100]     axis max value
 * @returns {string} inline SVG
 */
export function renderRadar(axes, { size = 320, max = 100 } = {}) {
  const cx = size / 2, cy = size / 2, r = size / 2 - 40;
  const n = axes.length;

  // Axis endpoints
  const angle = (i) => (2 * Math.PI * i / n) - Math.PI / 2;
  const point = (i, v) => [
    cx + r * (v / max) * Math.cos(angle(i)),
    cy + r * (v / max) * Math.sin(angle(i)),
  ];

  // Concentric grid rings (4 rings at 25/50/75/100%)
  const rings = [0.25, 0.5, 0.75, 1.0].map(pct => {
    const pts = axes.map((_, i) => point(i, max * pct));
    return `<polygon points="${pts.map(p => p.join(',')).join(' ')}"
              fill="none" stroke="#e2e8f0" stroke-width="1"/>`;
  }).join('');

  // Axis lines
  const spokes = axes.map((_, i) => {
    const [x, y] = point(i, max);
    return `<line x1="${cx}" y1="${cy}" x2="${x}" y2="${y}"
              stroke="#cbd5e1" stroke-width="1"/>`;
  }).join('');

  // Data polygon
  const dataPts = axes.map((a, i) => point(i, a.value));
  const polygon = `<polygon points="${dataPts.map(p => p.join(',')).join(' ')}"
                     fill="#6366f1" fill-opacity="0.2"
                     stroke="#6366f1" stroke-width="2"/>`;

  // Data points + labels
  const points = dataPts.map((p, i) =>
    `<circle cx="${p[0]}" cy="${p[1]}" r="4" fill="#6366f1"/>`
  ).join('');

  const labels = axes.map((a, i) => {
    const [x, y] = point(i, max * 1.15);
    return `<text x="${x}" y="${y}" text-anchor="middle"
              font-size="13" font-family="system-ui,sans-serif"
              fill="#0f172a">${a.label}</text>
    <text x="${x}" y="${y + 16}" text-anchor="middle"
              font-size="11" font-family="system-ui,sans-serif"
              fill="#64748b">${a.value}%</text>`;
  }).join('');

  return `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 ${size} ${size}"
    width="100%" style="max-width:${size}px;">
    ${rings}${spokes}${polygon}${points}${labels}
  </svg>`;
}
```

### Section usage

```javascript
export function sectionEngineRadar(snapshots) {
```

## P5 — Domain-grouped citations

**Goal:** collapse individual URLs into domain-level rows with type classification. Competitive intelligence view.

**File:** `lib/report/sections.js` → rewrite `sectionCanonicalSources`.

### URL-to-type heuristics

Simple regex map (no network calls):

```javascript
const DOMAIN_TYPE = [
  { re: /\bdirectory|listings?|catalog/i, type: 'directory' },
  { re: /\bblog|post|article|story/i, type: 'blog' },
  { re: /news|press|media/i, type: 'news' },
  { re: /github\.com|gitlab\.com/i, type: 'code' },
  { re: /npmjs\.com|pypi\.org|packagist/i, type: 'package' },
  { re: /reddit\.com|hackernews|ycombinator/i, type: 'community' },
  { re: /wikipedia\.org|wikidata/i, type: 'reference' },
  { re: /stackoverflow|stackexchange/i, type: 'Q&A' },
  { re: /maps\.google|goo\.gl\/maps/i, type: 'maps' },
];

function classifyUrl(url) {
  for (const { re, type } of DOMAIN_TYPE) if (re.test(url)) return type;
  // Default: if root domain ends in .com/.io/.co → 'site'
  return 'site';
}

function extractDomain(url) {
  try {
    return new URL(url).hostname.replace(/^www\./, '');
  } catch { return url; }
}
```

### Grouping

Collapse all citations to same domain into single row showing count:

```javascript
function groupByDomain(citations) {
  const groups = {};
  for (const c of citations) {
    const domain = extractDomain(c.url);
    if (!groups[domain]) {
      groups[domain] = { domain, count: 0, type: classifyUrl(c.url), urls: [] };
    }
    groups[domain].count += c.count || 1;
    groups[domain].urls.push(c.url);
  }
  return Object.values(groups).sort((a, b) => b.count - a.count).slice(0, 10);
}
```

### Render

```markdown
## Pages AI Engines Cite

{barchart rendering domain + type tag + count, e.g.:}

aeodirectory.com    [directory]   ━━━━━━━━ 5
jonnylangefeld.com  [blog]        ━━ 1
minuttia.com        [agency/site] ━━ 1
```

Barchart rows show: `<domain>` in bold, `[<type>]` in muted color, count right-aligned.

### Acceptance

- 15 individual URLs from same domain → 1 row with count=15
- Type tag appears next to domain
- Top 10 domains shown
- Still sorted by count descending

## Deferred from this spec (noted for v0.5.2+)

Ideas that came up but didn't make v0.5.1 cut:

- **Animated SVG transitions** — week-over-week animation. Requires JS, breaks zero-deps. v0.7+ if ever.
- **Favicon fetching for domain rows** — needs network call, violates offline-report principle.
- **Interactive collapsible sections** — needs JS. Inline markdown stays flat.
- **Per-query confidence scoring** — "Q1 has stable detection, Q2 is flaky" — requires 3+ runs historical data. v0.6 when we have users.
- **Multi-language report output** — report stays English. Query generation already language-aware (Polish site → Polish queries) but the surrounding chrome stays English.
